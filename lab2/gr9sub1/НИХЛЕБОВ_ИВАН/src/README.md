## Описание
Супервизор порождает несколько воркеров, следит за ними и управляет их режимами работы с помощью сигналов:
- `SIGTERM` / `SIGINT` — корректное завершение ("graceful shutdown")
- `SIGHUP` — перезапуск воркеров ("graceful reload")
- `SIGUSR1` — переключение воркеров в "лёгкий" режим
- `SIGUSR2` — переключение воркеров в "тяжёлый" режим
- `SIGCHLD` — обработка завершения дочерних процессов и перезапуск при авариях

## Требования
- Linux (Ubuntu/Debian, WSL2 или виртуалка)
- Python 3.8+
- Утилиты: `ps`, `kill`, `pidstat`

## Запуск
Перейдите в папку с проектом и выполните:
```bash
python3 src/supervisor.py
После запуска появятся строки:
[Supervisor] started
[Worker 1234] started in heavy mode
[Worker 1235] started in heavy mode
...
Управление супервизором

Откройте второй терминал и используйте PID супервизора:
Посмотреть процессы:ps -f --ppid <PID_супервизора>
Переключить воркеров в лёгкий режим:kill -SIGUSR1 <PID_супервизора>
Переключить обратно в тяжёлый режим:kill -SIGUSR2 <PID_супервизора>
Перезагрузить воркеров:kill -SIGHUP <PID_супервизора>
Завершить все процессы:kill -SIGTERM <PID_супервизора>

Диагностика

Проверить нагрузку воркеров можно так:
pidstat -H -p <PID_воркера>
Демонстрация

Мини-сценарий для проверки:

Запустить supervisor.py

Проверить список воркеров через ps

Отправить SIGUSR1, увидеть "switched to light mode"

Отправить SIGUSR2, увидеть "switched to heavy mode"

Отправить SIGHUP, увидеть перезапуск воркеров

Отправить SIGTERM, убедиться в корректном завершении
