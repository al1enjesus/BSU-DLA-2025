# Лабораторная 3

## Описание

Реализация утилиты `pstat` для чтения метрик процессов из `/proc` и сравнения с системными утилитами.

### Требования
- GCC (или совместимый компилятор C)
- Linux с файловой системой `/proc`
- Опционально: `sysstat` (для `pidstat`), `strace`, `perf`

### Сборка
```bash
make
```

### Очистка
```bash
make clean
```

### Полная демонстрация
```bash
./run.sh
```


## Что показывает pstat

Утилита читает следующие файлы из `/proc/<pid>/`:
- `stat` - основная статистика (PPid, состояние, время CPU)
- `status` - расширенная информация (потоки, память, переключения контекста)
- `smaps_rollup` - детали памяти (RssAnon, RssFile) - если доступно
- `io` - статистика ввода/вывода - если доступно


## Ограничения

- `/proc/<pid>/io` может требовать права root
- `/proc/<pid>/smaps_rollup` доступен только в новых ядрах Linux (>= 4.14)
- На WSL2 могут быть ограничения для некоторых файлов `/proc`
- `perf` может быть недоступен в контейнерах или требовать дополнительных прав


## Зависимости для полной демонстрации

```bash
# Ubuntu/Debian
sudo apt install build-essential sysstat strace linux-tools-generic

# CentOS/RHEL
sudo yum install gcc make sysstat strace perf
```
