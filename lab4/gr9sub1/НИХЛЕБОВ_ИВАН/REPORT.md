# Отчет по лабораторной работе №4
## Анализ системных вызовов и производительности команд ls, cat, grep

**Студент:** №16  
**Группа программ:** `ls`, `cat`, `grep`

---

### 1. Цель работы
Изучить поведение типичных утилит Linux (`ls`, `cat`, `grep`) с точки зрения системных вызовов и использования CPU/IO, используя утилиты `strace` и `perf`.

### 2. Инструменты
- `strace -c` — сбор статистики системных вызовов.
- `perf stat` — измерение производительности (циклы, инструкции, кэш-промахи и т.д.).

### 3. Проведенные эксперименты
Для каждой программы были запущены команды:
```
strace -c <программа>
perf stat <программа>
```

Программы:
- `ls` — получение списка файлов.
- `cat /etc/hostname` — чтение текстового файла.
- `grep test logs/tmpfile.txt` — поиск подстроки в файле.

### 4. Результаты наблюдений
- `ls`: выполняет системные вызовы, связанные с чтением каталогов (`getdents64`, `stat`), затраты CPU невелики.  
- `cat`: обращается к системным вызовам `openat`, `read`, `write`, зависит от размера файла.  
- `grep`: использует больше CPU, чем `cat`, из-за обработки текста и поиска совпадений.

### 5. Выводы
1. Для простых программ доля системных вызовов велика по сравнению с пользовательским кодом.  
2. `grep` демонстрирует большую нагрузку на CPU за счет анализа текста.  
3. `strace` подходит для изучения IO, `perf` — для анализа производительности.

---
**Итог:** получены практические навыки анализа системных вызовов и производительности процессов.
