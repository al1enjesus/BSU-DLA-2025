## Анализ системных вызовов и производительности команд ls, cat, grep

 
**Группа программ:** `ls`, `cat`, `grep`

---

### 1. Цель работы
Изучить поведение типичных утилит `ls`, `cat`, `grep` с использованием CPU/IO, используя утилиты `strace` и `perf`.

### 2. Инструменты
- `strace -c` — сбор статистики системных вызовов.
- `perf stat` — измерение производительности (циклы, инструкции, кэш-промахи и т.д.).

### 3. Проведенные эксперименты
Для каждой программы были запущены команды:
```
strace -c <программа>
perf stat <программа>
```

Программы:
- `ls` — получение списка файлов.
- `cat /etc/hostname` — чтение текстового файла.
- `grep test logs/tmpfile.txt` — поиск подстроки в файле.

### 4. Результаты 
- `ls` - выполняет системные вызовы, связанные с чтением каталогов `getdents64`, `stat`.  
- `cat` - обращается к системным вызовам `openat`, `read`, `write`.  
- `grep` - использует больше CPU, чем `cat`.

### 5. Выводы
1. Для простых программ доля системных вызовов велика по сравнению с пользовательским кодом.  
2. `grep` демонстрирует большую нагрузку на CPU путем анализа текста.  
3. `strace` - для изучения IO, `perf` — для анализа производительности.
